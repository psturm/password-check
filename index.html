Password entropy

Entropy is another way of describing the number of guesses needed to find the correct password.

The entropy in bits is basically the base-2 logarithm of that number
(e.g. 23 bit: 2^23 = 8,388,608 guesses).

In average, an attacker only has to guess half that number to find the password.

Tips

Password selection is always depending on your "threat model", but here are some general guidelines and tips:

  • use long passwords, minimum 12 characters (more characters = better)
  • use symbols of all kinds (lowercase, uppercase, digits, special characters)
  • don't use names or words from dictionaries
  • don't reuse a password on multiple sites

Ideally, use a password manager software, e.g. Keepass and generate a random password for each site.

F.A.Q.

How much entropy is "safe"?
This depends on a lot of factors and can't be easily answered. Modern hardware or even hardware clusters can brute force passwords pretty quickly nowadays. How fast is only matter of effort (money).
